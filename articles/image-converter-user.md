## Which Format Should You Use?

**JPG** — Best for photographs and complex images. Lossy compression keeps file sizes small. No transparency support.

**PNG** — Best for screenshots, logos, and images with text. Lossless compression. Supports transparent backgrounds.

**WebP** — Best for web use. Smaller file sizes than both JPG and PNG at similar quality. Supported in all modern browsers.

Knowing which format to pick saves bandwidth, preserves quality, and avoids the blank-background problem (saving a transparent PNG as JPG fills it white).

## Common Use Cases

**Removing white backgrounds from JPG logos**
JPGs don't support transparency. If you have a logo saved as JPG with a white background that you want transparent, convert it to PNG first, then remove the background in an editor.

**Optimising images for a website**
WebP files are 25–35% smaller than equivalent JPGs. Convert your image library to WebP to reduce page load times without visible quality loss.

**Social media format requirements**
Some platforms only accept certain formats. Twitter and Instagram handle JPG and PNG; some ad platforms require one or the other. Convert once and upload.

**Sending screenshots via email**
Screenshots are often PNG (large files). Converting to JPG reduces size for attachments while keeping the image readable.

**Converting WebP images you downloaded**
WebP is browser-native but not universally supported by older apps, design tools, or CMS platforms. Convert to JPG or PNG for maximum compatibility.
