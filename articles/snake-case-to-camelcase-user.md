## The 8 Case Formats Developers Actually Use

**snake_case** тАФ Python variables and functions, database column names, environment variable names. All lowercase, words joined by underscores. Constants use `SCREAMING_SNAKE_CASE` (all caps).

**camelCase** тАФ JavaScript and TypeScript variables, JSON keys, Java and Kotlin methods. First word lowercase, each subsequent word capitalized. No separators.

**PascalCase** тАФ Class names across most languages, React components, TypeScript interfaces and types, C# everything. Same as camelCase but the first word is also capitalized.

**kebab-case** тАФ URL slugs, CSS class names, HTML data attributes, npm package names. Cannot be used as a variable name in most languages because the hyphen is parsed as a minus operator.

**SCREAMING_SNAKE_CASE** тАФ Constants and environment variables. `API_BASE_URL`, `MAX_RETRY_COUNT`, `.env` file keys.

**Title Case** тАФ Document headings, article titles, product names. "The Quick Brown Fox". Not used in code identifiers.

**lowercase / UPPERCASE** тАФ Normalization before comparison, CSS text transforms, database IDs.

**Renaming across codebases** тАФ Moving a module from one language to another means every identifier needs a naming convention change. Finding and replacing by hand misses things.
